# Naming R2 — Ranked Top 12 (OSS LLM+MCP Gateway, Rust, single binary, agent-first)

Scoring rubric, ruthlessly applied:
1. **Types cleanly** as a CLI command / binary / org
2. **Pronounceable on first sight** by a non-native English speaker
3. **Sounds like infrastructure**, not a SaaS app (envoy / caddy / vector / temporal register)
4. **Memorable**
5. **Low collision risk** per the agents' findings

Hard `collision` verdicts excluded across the board (Velo, Klave, Splice, Crossbar, Conduit, Freeport, Wicket, Spillway/flow-current, Flume, Sluice, Runnel, Voxen, Aptix, Velar, Rova, Wend, Vant). None of those were "truly minor."

## Ranked Top 12

### 1. Busbar
**Rationale:** The single conductor bar every circuit taps into — a near-perfect metaphor for one binary every LLM/MCP call routes through. Two hard syllables, unmistakable spelling, reads as genuine infrastructure.
**Collision:** Clean. Only electrical-CAD software shares the word; zero AI/devtool/npm/crate/PyPI overlap. The EE meaning signals "wiring," an asset not a competitor.
**Verify:** crates.io `busbar`, npm `busbar`, PyPI `busbar`, GitHub org `busbar`, busbar.dev / .io / .sh, TM class 9/42.

### 2. Throughway
**Rationale:** The road every unit of traffic goes through — reads instantly as a gateway, one strong compound, types clean. Safest serious-infra pick with the most immediate meaning.
**Collision:** Clean. No software/npm/crate/PyPI/AI-company namesake; only generic "AI gateway" category pages.
**Verify:** crates.io `throughway`, npm `throughway`, PyPI `throughway`, GitHub org `throughway`, throughway.dev / .io / .sh.

### 3. Tyne
**Rationale:** Four letters, one syllable, coined feel, types beautifully as `tyne`. Clean phonotactics, easy to say once and remember, sits naturally next to caddy/vector/temporal.
**Collision:** Clean. No AI/dev-tool/infra/package collision; only generic UK-river geography results, none in the LLM/gateway space.
**Verify:** crates.io `tyne`, npm `tyne`, PyPI `tyne`, GitHub org `tyne`, tyne.dev / .io / .sh, TM check (geographic term, easy to clear in software class).

### 4. Postern
**Rationale:** A small secondary gate in a fortress wall — a genuine gateway word, infrastructural in tone, typeable as `postern serve`, pronounceable on first sight. Quiet and serious like envoy/caddy.
**Collision:** Clean. No AI-gateway hits, no crate/npm/PyPI package, no prominent repo.
**Verify:** crates.io `postern`, npm `postern`, PyPI `postern`, GitHub org `postern`, postern.dev / .io / .sh.

### 5. Queryx
**Rationale:** `query` + x — reads instantly as a request/LLM gateway, types clean (`queryx serve`, `queryx route`). The most on-concept coined name with an open namespace.
**Collision:** Clean. No notable software/AI company, npm, crate, or prominent repo. Soft risk only: "query" reads slightly generic/DB-adjacent.
**Verify:** crates.io `queryx`, npm `queryx`, PyPI `queryx`, GitHub org `queryx`, queryx.dev / .io / .sh.

### 6. Relo
**Rationale:** Warm, vowel-friendly, two-syllable mouthfeel; reads as relay/re-route (directly on-theme); terminal-clean and instantly typable. Easiest non-native pronunciation of the coined set.
**Collision:** Clean. WebSearch surfaced no namesake AI/dev product, company, or prominent package — only generic AI-gateway results.
**Verify:** crates.io `relo`, npm `relo`, PyPI `relo`, GitHub org `relo`, relo.dev / .io / .sh (short .com likely taken — check).

### 7. Korv
**Rationale:** Four letters, hard consonant edges read as serious infra (cf. Kong, Caddy). Distinctive, instantly typable, ownable as `korv` org/binary.
**Collision:** Clean. No software/AI/dev-tool or package collision; nearest neighbor is unrelated Korbit AI. Flag: "korv" = Swedish for sausage (unlikely to matter to an EN dev audience).
**Verify:** crates.io `korv`, npm `korv`, PyPI `korv`, GitHub org `korv`, korv.dev / .io / .sh.

### 8. Spillway
**Rationale:** The engineered channel controlling overflow from a dam — connotes controlled high-volume throughput under pressure (rate limits, fallback, load shedding). Clean compound, typable, infra-serious.
**Collision:** Minor. The plain-english agent found no name collision (only generic gateway packages). NOTE: a separate flow-current agent flagged `luhn/spillway`, a dockerized load-shedding reverse proxy — small, no crate/npm, but same adjacent category. Verify before committing; ranked below the cleaner picks for that reason.
**Verify:** crates.io `spillway`, npm `spillway`, PyPI `spillway`, GitHub `spillway` (check luhn/spillway activity), spillway.dev / .io / .sh.

### 9. Stile
**Rationale:** Steps that let you cross a wall/fence — a literal passage-through. Five letters, instantly typable, clean, modern, brandable. Easy first-sight pronunciation.
**Collision:** Minor. Only a tiny dormant npm package `stile` (a React inline-style helper); no AI/infra overlap, no Rust crate, no prominent project.
**Verify:** crates.io `stile`, npm `stile` (taken-but-dormant — consider scope or binary-only), PyPI `stile`, GitHub org `stile`, stile.dev / .io / .sh.

### 10. Freshet
**Rationale:** The surge of fresh water that floods a stream — a flow-burst word, distinctive and almost entirely unclaimed. Cleanest semantic namespace in the flow direction.
**Collision:** Minor. Only one dormant repo `FreshetDMS/Freshet` (inactive stream-processing prototype); no crate, npm, PyPI, or live company.
**Verify:** crates.io `freshet`, npm `freshet`, PyPI `freshet`, GitHub org `freshet`, freshet.dev / .io / .sh.

### 11. Weir
**Rationale:** A low dam that regulates and measures a river's flow — the single control point every unit of flow passes over. Ideal gateway metaphor, hydraulic-engineering pedigree, 4 chars, never misspelled.
**Collision:** Minor. Package namespaces clean (no `weir` crate, no prominent npm/GitHub gateway). Friction is brand/.ai only: WEIR AI (weir.ai) is a ~7-person identity/likeness startup in a different category.
**Verify:** crates.io `weir`, npm `weir`, PyPI `weir`, GitHub org `weir`, weir.dev / .io / .sh (avoid .ai — taken; TM check vs WEIR AI).

### 12. Narthex
**Rationale:** The entrance/threshold hall of a building — the literal way in. Distinctive, modern-sounding, fully ownable as a brand; reads as infra not SaaS.
**Collision:** Minor. No AI-gateway collision; a few small unrelated GitHub repos (metadata tool, dictionary generator) but none prominent or in-space. Slightly harder first-sight pronunciation than postern/stile — the reason it sits at #12.
**Verify:** crates.io `narthex`, npm `narthex`, PyPI `narthex`, GitHub org `narthex`, narthex.dev / .io / .sh.

## Notable cuts (and why they missed)
- **Trunkline / Switchyard / Transom / Gangway / Sallyport** — solid meaning, but each carries an in-or-adjacent-segment shadow (Trunk.io, dormant SOA/k8s tools, TransomJS, Enarx crate). Beaten by cleaner picks.
- **Quell / Naxa / Klyro / Vesp** — Quell softly bends the "no meaning" bar; Naxa/Klyro/Vesp carry real package-scope or company SEO friction and weaker infra register.
- **Kryos / Relyx / Portix / Voltic** — coined and pronounceable but each has SEO or org-name contention; none beat Queryx for on-concept fit.
- **Rivo / Oxin / Inlet / Soko** — oximy-adjacent but contested (Rivo.io funded, Oxen.ai homophone, Inlet AI + common-word search noise, `soko` npm taken).
- **Shunt** — semantically backwards (divert AWAY, not flow THROUGH) despite clean namespace.
